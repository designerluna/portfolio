IMS Design System

Redesigning the Indianapolis Motor Speedway website and building a robust, accessible component library.

Client Indianapolis Motor Speedway
My Role UI Designer
Duration 3 Months
Platform Figma & Web

01. Overview

Redesigning a Racing Legend

The team at the Indianapolis Motor Speedway (IMS) contracted our team to redesign their website and create an accessible, responsive, and user friendly design system. The Indianapolis Motor Speedway website consists of event details, online ticket purchases, event schedules, and other helpful information related to the Indianapolis Motor Speedway.

The team consisted of a Web Developer, UX Designer, and myself as the UI Designer.

Objectives

  • Create a robust component library.
  • Simplify an extensive navigation system.
  • Create easy-to-use Figma templates so the IMS Marketing team can create page mockups independently.
  • Train the IMS team on accessibility best practices when using the components.

Team Roles & Responsibilities

UX Designer

  • Gather requirements for new website and component library.
  • Lead brainstorming sessions with clients.
  • Create initial wireframes of new website concepts.
  • Assist with the creation of design system on Figma.

UI Designer (Myself)

  • Use Figma to create components and their variants.
  • Ensure all new components meet all accessibility requirements.
  • Collaborate with developer to ensure Figma components are properly translated to coded library.

Web Developer

  • Use new Figma component library to develop components to be used by IMS developers.
  • Redesign website using new component library and Figma wireframes.
