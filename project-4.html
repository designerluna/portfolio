Cincinnati Children's Hospital ARIA app

Creating an AI-powered iPad application to identify and assess risk levels in children.

Client Cincinnati Children's
My Role UX/UI Designer
Duration 3 Months
Platform iPadOS (Tablet)

01. Overview

Identifying Risk via Audio AI

The Cincinnati Children's Hospital was seeking a digital solution to help them pinpoint children who are at risk for developing or currently have violent tendencies.

My team and I were contracted to create an iPad application called ARIA that therapists could use to record audio from interviews with students. At the end of the recording, the audio would be uploaded and transcribed. With the help of AI, the transcriptions would be used to categorize the student's risk level as low, medium, or high. Therapists could then take action to help students depending on their risk level.

The project team consisted of a project manager, two developers, and myself as the UX/UI SME.

Team Responsibilities

UI Developer

  • Attended regular check-in meetings with the UX/UI Designer

02. Wireframes

Figma Wireframes & User Stories

To start, the project manager shared user stories that would help us plan an MVP for the application. We categorized the requested features for the app into 3 distinct tabs: Students, Reports, and Administration.

1. Login Flow

The first screen that will display when the ARIA app is launched is the login screen. This screen shows a standard login experience where the user can enter their email and password. Within this Figma component, I included various properties of errors that the developers could reference when developing error states.

Figma Login View with custom validation logic

2. Students and Interviews Tab

Once the user is logged in, they will see a page where they can either search for a student or add a new student. I learned that upon first launch, a user would want to quickly find the information of a student in the database, or to add a new student so an interview can be conducted. Therefore, it made sense to make this the landing page.

ARIA Student Search Landing Page

If the user decides to search for a student, they can fill out any of the 5 fields to refine their search. In the example below, the user searched by first name, last name, and gender. Since the number of results will eventually get lengthy as the app ages, we included a sorting feature in the table of results. Users can use the arrow buttons in the table headings to sort the results and then click on a row when they've found the student they're searching for.

Search Results Grid with sortable headers

The Search Students Results page also includes the Add New Student button just in case the user cannot find a student they may have thought to be in the database. This will save them time so they don't have to return to the landing page to add the student. After clicking the Add New Student button, a fullscreen modal will display. Within the modal the user can enter the new student's information. The user can click Save once the required fields have been successfully filled out.

Add New Student Fullscreen Modal

The following screen shows the Student Details page. In this example, the student has already completed multiple interviews. Here the user can edit the student's demographic information, start a new interview, sort through a list of completed interviews, or click on one of the existing interviews to see more information regarding the selected interview.

Student Details & Completed Interviews page

If a user chooses to view an existing interview, they will see the following screen, which shows the interview details and a button to download the interview audio. They can also see saved notes and add a new note as well. At the top, there is a button they can click to quickly start a new interview for the student.

Completed Interview details page

If the user chooses to start a new interview, they see a similar screen to the last one, but it does not yet include any interview details, notes, and mp3 download link. When the user is ready to start the recording, they can click the Start Interview button. When they are ready to end the recording, they can click the Complete Interview button.

New Interview recording controls page

The following screen shows the loading popup that would display after the interviewer clicks Complete Interview.

Audio upload and AI transcription progress modal

3. Reports Tab

An important feature that was requested by the stakeholder was the ability to export student details and interview results. This feature has a specific purpose, therefore I made it a separate tab within the app. The user can filter their export, and once they have the list of results they want, they can click the export dropdown and select their preferred file type for the download.

Export manager and reporting filters tab

4. Administration Tab

Another important feature that was requested was a page where admins can add a new user or admin, depending on their preferred level of permissions. This feature would show or hide depending on the level of permissions an account has. Due to this, it made sense to have this feature in it's own tab.

Administration account list and permissions overview

The following page would display when an admin selects an existing user from the list of users. When viewing an existing user, the 3 fields on the page would be populated with the capability to edit, save, and delete user.

Existing User field management modal

The following fullscreen modal would display when an admin selects Add New User. They would be able to save once the 3 fields have been filled out.

Add New User modal fields

The following page would display when an admin selects Add New User, configuring specific credentials and access settings.

Add New Admin User credential configuration view

03. Conclusion

Successful Launch & Key Learnings

This application was successfully launched and is now being used by qualified therapists. The client was satisfied with the new application.

This was my first time designing an application that was solely meant to be used on an iPad. I learned how to use iPadOS resources for designers, such as Figma iPadOS templates and SF symbols.
